# Шаблон веб-интерфейса для браузерной игры

## Структура проекта

```
/workspace/
├── index.html          # Главный файл с авторизацией и основным каркасом
├── css/
│   └── style.css       # Все стили интерфейса
├── js/
│   └── app.js          # Логика навигации и управления состоянием
└── pages/
    ├── profile.html    # Страница настроек профиля
    ├── shop.html       # Страница магазина
    ├── arsenal.html    # Страница арсенала
    ├── friends.html    # Страница друзей
    └── chat.html       # Страница чата
```

## Особенности шаблона

### Адаптивность
- Интерфейс адаптируется под ПК и мобильные устройства
- На мобильных устройствах занимает 100% экрана
- На ПК отображается в центре с соотношением сторон 16:7 (80% ширины экрана)
- Все кнопки оптимизированы для нажатия пальцем (минимальная высота 45-50px)

### Динамическая загрузка
- Страницы загружаются динамически через fetch() без перезагрузки
- Каждая страница находится в отдельном файле в папке pages/
- Данные не загружаются пока пользователь не перейдет в соответствующий раздел

### Прокрутка
- Реализована там, где есть списки (персонажи, предметы, способности, чат, друзья)
- Скроллбар стилизован под общий дизайн

## Интеграция с бэкендом

### 1. Авторизация
В файле `js/app.js` найдите функции:
- `handleLogin(event)` - обработка входа
- `handleRegister(event)` - обработка регистрации

Замените console.log на AJAX/Fetch запросы к вашему API.

### 2. Загрузка данных
Для загрузки данных с сервера измените функции в `js/app.js`:
- `loadFactions()` - загрузка фракций
- `loadCharacters(factionId)` - загрузка персонажей
- `loadItems()` - загрузка предметов
- `loadFriendsList()` - загрузка списка друзей
- `loadFriendRequests()` - загрузка запросов в друзья
- `loadChatMessages()` - загрузка сообщений чата

### 3. PHP интеграция
Вы можете заменить статические данные в файлах pages/*.html на PHP код:

Пример для shop.html:
```php
<?php
// Вместо статического массива factions
$factions = $db->query("SELECT * FROM factions")->fetchAll();
foreach ($factions as $faction): ?>
    <div class="faction-card" onclick="loadCharacters(<?= $faction['id'] ?>)">
        <div class="faction-name"><?= htmlspecialchars($faction['name']) ?></div>
        <div class="faction-desc"><?= htmlspecialchars($faction['description']) ?></div>
    </div>
<?php endforeach; ?>
```

### 4. Сохранение состояния
Объект `GameState` в `js/app.js` хранит текущее состояние клиента.
При загрузке страницы вы можете заполнить его данными из сессии/БД.

## Стилизация

Все CSS переменные находятся в `:root` в начале файла `css/style.css`:
- `--primary-color` - основной цвет
- `--secondary-color` - вторичный цвет (синий)
- `--accent-color` - акцентный цвет (красный)
- `--text-color` - цвет текста
- `--bg-color` - цвет фона
- `--panel-bg` - фон панелей

Измените их для быстрой смены цветовой схемы.

## Тестирование

1. Откройте index.html в браузере
2. Введите любые данные для входа (валидация только клиентская)
3. После входа протестируйте все разделы:
   - Магазин (персонажи, предметы, валюта)
   - Арсенал (выбор персонажа, сокеты)
   - Друзья (поиск, запросы)
   - Чат (отправка сообщений)
   - Профиль (настройки, статистика)

## Мобильное тестирование

Для тестирования мобильной версии:
1. Откройте DevTools в браузере (F12)
2. Включите эмуляцию мобильного устройства
3. Проверьте удобство нажатия кнопок и читаемость текста
